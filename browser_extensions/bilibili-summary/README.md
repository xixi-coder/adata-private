# Bilibili Subtitle TXT Extension

这个目录是一个 Chrome/Edge Manifest V3 插件：打开 B 站视频页后，页面右下角会出现“字”按钮，可读取官方字幕并下载为 `.txt` 文件。

## 加载插件

1. 打开 `chrome://extensions/` 或 `edge://extensions/`
2. 开启开发者模式
3. 选择“加载已解压的扩展程序”
4. 选择这个目录：

```text
/Users/xixi/pythonProject/adata/browser_extensions/bilibili-summary
```

## 使用

1. 打开 B 站视频页
2. 点击右下角“字”按钮
3. 点击“读取字幕”预览文本，或直接点“下载 TXT”

## 原理

插件完全在浏览器里运行，不需要本地服务，也不需要 API key。

核心流程：

1. `content.js` 注入 B 站视频页，在页面右下角渲染导出面板。
2. 插件从当前 URL 和页面初始化数据中读取 BV/AV 号、标题、分 P 等信息。
3. 如果页面里没有直接给出 `cid`，插件会调用 B 站公开视频信息接口 `https://api.bilibili.com/x/web-interface/view` 补全 `cid`。
4. 拿到 BV/AV 号和 `cid` 后，优先请求播放器字幕接口 `https://api.bilibili.com/x/player/v2`，读取官方字幕或 AI 字幕列表。
5. 如果普通字幕接口没有返回字幕列表，`hook.js` 会在页面主环境里监听播放器自己的 `fetch` / `XMLHttpRequest` 请求，捕获形如 B 站字幕 JSON 的响应。
6. 找到字幕 JSON 后，插件把其中的 `body[].from` 和 `body[].content` 转成带时间戳的纯文本。
7. 点击“下载 TXT”时，浏览器用 `Blob` 生成本地 `.txt` 文件，不经过任何后端。

当前有三层兜底：

- 官方字幕接口：适合大多数普通视频。
- 播放器请求捕获：适合播放器里能看到 AI 字幕，但接口列表为空的情况。
- 当前屏幕字幕：如果完整字幕抓不到，会尽量读取播放器当前可见字幕，用于确认字幕 DOM 是否可访问。

## 如何判断字幕来源

点击“读取字幕”后，面板状态栏会显示本次使用的来源；下载的 TXT 顶部也会写入 `字幕来源`、`字幕语言` 和 `来源详情`。

可能的来源：

- `官方字幕接口`：通过 `x/player/v2` 拿到字幕列表，再请求对应的字幕 JSON，最完整也最推荐。
- `播放器请求捕获`：`hook.js` 捕获到播放器运行时自己请求的字幕 JSON，常见于播放器能显示 AI 字幕但接口列表为空的页面。
- `已加载资源扫描`：从浏览器已经加载过的资源或页面 HTML 里扫描到字幕 JSON 地址。
- `当前屏幕字幕`：只读到了播放器当前可见的一句字幕，不是完整字幕文件，通常说明前几种路径都没有拿到完整 JSON。

- 优先读取 B 站官方字幕，包括部分 AI 字幕。
- TXT 内容包含标题、链接、BVID/AID/CID、分 P、字幕来源和带时间戳的字幕。
- 下载文件名会包含标题、BV/AV 号、分 P、CID 和导出时间，避免不同视频互相覆盖。
- 没有字幕的视频会提示暂无字幕；音频下载和 Whisper 转写不在这个轻量版里。
