# 持仓组合分析 Skill

对一个账户做**完整闭环**的分析：组合诊断 → 个股深挖 → 大盘与消息面 → 可执行操作建议。不只是算权重，而要回答："这个组合押在什么上、押得多重、每只票现在该怎么办、在当前大盘和消息环境下如何调整。"

## 核心原则

1. **数据驱动，不编造**：权重、盈亏、占比、财务、行情等必须由持仓数据或真实接口/检索结果算出。拿不到的数据标注「缺失」，不臆造数字。
2. **闭环分析**：组合结构、个股基本面/技术面/资金面/消息面、大盘环境、操作建议，一个都不少。个股环节的方法论对齐 `stock-analysis` skill。
3. **风险优先**：先讲清组合系统性风险（集中、相关、回撤）和大盘风险，再谈机会与操作。
4. **给可执行的操作建议，但讲清前提和风险**：允许给出明确方向（加仓 / 减仓 / 持有 / 止盈 / 止损 / 观望）并附**参考区间**和**触发条件**（如"跌破 X 或放量滞涨可减仓""回踩 Y 附近企稳可关注"）。但必须说明这是分析师视角的参考而非确定性指令、不承诺收益、风险自担，结尾附免责声明。避免"必涨 / 保证 / 满仓梭哈"这类话术。

## 输入处理

用户可能以三种方式提供持仓：
- **持仓截图**（最常见，如同花顺、券商App 的持仓页）。
- 文本/表格形式的持仓列表。
- 只报几个股票名称 + 大致仓位。

### 从截图解析持仓（重点）

逐行、逐列读取，常见列含义：
- **股票名称**：注意港股会有「沪港/港股」标记或代码前带 `HK`；美股同理。
- **市值**：该持仓当前总市值（通常是每行第一个大数字）。
- **盈亏 / 盈亏%**：浮动盈亏金额与百分比（红涨绿跌，注意正负号）。
- **持仓 / 可用**：持有股数 / 可卖股数。
- **成本 / 现价**：成本价 / 最新价（港股会带 `HK$`，美股带 `$`）。

解析注意事项：
- **区分币种**：A股为人民币，港股 `HK$`、美股 `$`。做组合合计与占比时，若含外币持仓，统一说明是否已折算；截图市值通常已折成人民币，可直接相加，但要在结论里注明这一假设。
- **区分品种**：股票、可转债（名称常含「发债 / 转债」）、ETF/基金要分类。可转债、货基等低风险仓位单独归为一类，不混入股票赛道敞口。
- **数字校验**：市值 ≈ 现价 × 持股数；盈亏 ≈ (现价 − 成本) × 持股数。用这两条关系交叉验证识别是否正确，发现明显对不上就标注存疑。
- 识别不清的字段明确列出「以下数据未能识别，请补充」，不要猜。

## 分析流程

### 1. 建立持仓总表
把每一行整理成结构化表格：名称 / 代码（若能识别或查得）/ 市值 / 占比 / 浮盈亏 / 盈亏% / 所属赛道或行业。
计算：
- **股票总市值**、**非股票（债/基金）市值**、**账户合计**。
- 各持仓 **占比（权重）** = 市值 / 合计。
- **组合总浮盈亏** = Σ 各持仓浮盈亏；**整体成本** = 合计市值 − 总浮盈亏；**整体浮盈率**。

### 2. 集中度分析
- **单票最大权重**、**前 3 / 前 5 大持仓合计占比**。
- 是否存在单票过重（如 >20%）或头部过度集中（前 3 > 50%）。
- 可选：用简单口径衡量分散度（如前 N 大集中度、赛道数量）。

### 3. 赛道 / 行业敞口
- 把每只股票归入行业或投资主题（可用 `adata` 的申万行业接口辅助，见下）。
- **按主题汇总权重**：把同一条产业链/主题的持仓加总（例如"PCB""光模块""半导体设备""内存接口"都属"AI 算力链"）。
- 判断组合是否押注在**单一主线**上——高度正相关的持仓会同涨同跌，放大回撤，这是组合最需要警惕的系统性风险。
- 给出敞口分布（主题占比排序），点明最大的集中来源。

### 4. 盈亏归因
- 分离**盈利贡献**与**亏损拖累**：列出赚钱的票（贡献了多少浮盈）和亏钱的票（拖累多少）。
- 指出浮盈是否高度集中在少数标的（"利润未落袋"风险）。
- 检查**仓位与盈亏状态是否错配**：例如最重仓压在浮亏/高成本票上，而大赢家仓位过轻。

### 5. 大盘与市场环境（必须，自上而下）
个股不能脱离大盘。用真实数据 + 检索判断当前市场环境：
- **指数位置与趋势**：上证综指、深证成指、创业板指、科创50，以及港股持仓相关的恒生指数/恒生科技。用 `adata` 指数行情看点位、均线、近期涨跌（见"数据获取参考"）。
- **市场情绪**：成交量能（放量/缩量）、涨跌家数、近期是普涨/普跌/结构分化。
- **板块轮动**：组合所在赛道近期是资金流入还是流出、是否处在风口或退潮（可用概念/行业资金流接口，取不到则用检索的市场综述）。
- **宏观与政策背景**：利率、汇率、重要会议/政策、外围市场（尤其影响 AI 算力链的美股科技股、英伟达等海外龙头动向）。
- 给出一句**大盘定调**：进攻 / 震荡 / 防守，作为后续操作建议的前提。

### 6. 个股深挖（对重点持仓逐一分析）
对**权重靠前**或**盈亏异常**（大幅浮盈需止盈、深套需复盘）的持仓，逐只做多维分析，方法论对齐 `stock-analysis` skill：
- **基本面**：营收/净利增速、毛利率/净利率、ROE、负债率、现金流质量（用 `adata` 财务接口 `get_core_index`）。
- **技术面**：均线排列、当前价相对成本与关键位、距高点回撤、量价关系（用 `adata` 行情接口）。
- **资金面**：主力资金近 N 日流向、放量缩量（接口不可用时标注缺失）。
- **消息面（结合实时检索）**：近期公告、业绩预告、行业景气变化、突发利好利空。**必须用 web 检索**获取最新消息，不能只靠记忆。
- 每只票给一句**个股定性**：逻辑是否还在、当前处于什么位置、主要矛盾是什么。

### 7. 消息面 / 时事（必须，用 web 检索）
- 检索组合核心赛道与重点个股的**最新新闻、公告、研报观点、行业数据**（如 AI 算力资本开支、算力/存储/PCB 景气、订单、扩产、政策）。
- 检索大盘与宏观的**近期重大事件**。
- 把检索到的信息**注明来源与日期**，并说明对相关持仓是利好还是利空。检索不到时说明，不编造。

### 8. 操作建议（核心产出，可执行）
在大盘定调的前提下，**逐只**给出操作建议，要具体、可落地：
- **动作**：加仓 / 减仓 / 持有 / 止盈（分批）/ 止损 / 观望，明确方向。
- **参考区间**：给出关注的价格区间或位置（用"参考位/关注区间"表述），如支撑位、压力位、止盈/止损参考价。
- **触发条件**：说明在什么情况下执行（如"若放量跌破 MA20 且赛道资金持续流出，可减仓部分锁盈""回踩 X 附近缩量企稳可关注加回"）。
- **组合层面再平衡**：结合集中度问题给出调整方向（如收敛单赛道敞口、对极端浮盈标的分批止盈、深套票按逻辑决定去留、把仓位向最有把握的标的集中）。
- 用**分档、条件化**的表述，而非单一确定性指令。区分短线（数日~数周）与中线（数月）视角。

### 9. 风险提示（必须）
- 组合当前主要下行风险 + 大盘系统性风险。
- 操作建议的前提假设与失效条件。
- 数据时效与识别局限（截图 OCR 可能有误、外币折算假设、资金/估值/港股数据可能缺失、检索信息时效等）。
- 免责声明。

## 输出格式

用中文，结构建议：

```
# 持仓组合分析与操作建议（数据日期）

## 一、组合概览
- 账户合计市值 / 总浮盈亏 / 整体浮盈率 / 持仓数量 / 品种构成

## 二、结构诊断
- 集中度（单票最大、前3/前5）
- 赛道/主题敞口（按主题汇总权重）

```

## 数据获取参考（adata 库）

组合分析主要基于用户提供的持仓数据；`adata` 用于**补充/校验**行业归类与最新价格。

```python
import adata

# 申万行业（用于赛道归类）
adata.stock.info.get_industry_sw(stock_code='002463')

# 全部A股代码表（用名称反查代码）
adata.stock.info.all_code()

# 最新/近期行情（校验现价、算近期回撤）
# A股默认百度源；科创板(688)等若百度返回空，切换东财：
adata.stock.market.set_market_provider('east', fallback=True)
adata.stock.market.get_market(stock_code='688008', k_type=1, start_date='2025-01-01')

# 个股深挖：财务核心指标（营收/利润/ROE/负债率/现金流）
adata.stock.finance.get_core_index(stock_code='002463')
adata.stock.info.get_stock_shares(stock_code='002463')   # 股本（估算市值/PE）

# 大盘环境：指数行情（上证/深成/创业板/科创50 等）
adata.stock.market.get_market_index(index_code='000001', k_type=1)  # 上证综指
# 常用指数代码：上证000001 深成399001 创业板399006 科创50 000688 沪深300 000300

# 资金流（受限网络下可能连不通，取不到标注缺失）
adata.stock.market.get_capital_flow(stock_code='002463')
```

**消息面 / 大盘环境必须结合 web 检索**（`adata` 不覆盖新闻/宏观）：检索个股与赛道的最新公告、业绩、研报、行业景气数据，以及大盘、宏观、外围（如英伟达等海外算力龙头）动向。检索结果注明来源与日期。

注意事项与本仓库实测经验：
- **行情数据源**：A股 K 线默认走百度，部分科创板/特殊标的百度会返回空，需 `set_market_provider('east', fallback=True)` 回退东财。
- **资金流接口**（`get_capital_flow`）依赖 `push2his.eastmoney.com`，在受限网络/代理环境下可能连不通，取不到时在结论中标注"资金流数据缺失"，不要编造。
- **港股/美股**：`adata` 主要覆盖 A 股，港股（如中芯国际 00981.HK）、美股的行业与行情可能取不到，按截图数据 + web 检索分析并标注数据来源限制。
- 接口返回空或报错，一律标注数据缺失，不用假数据填充。

## 注意事项

- 完整闭环：组合诊断 → 大盘环境 → 个股深挖 → 消息面 → 操作建议，环节不能只做前半段。
- 操作建议要**具体可执行**（动作 + 参考区间 + 触发条件 + 视角），但用分档、条件化表述，说明是参考而非确定性指令，不承诺收益、风险自担。
- 消息面和大盘环境**必须用 web 检索获取最新信息**，不能只凭记忆或用过期信息下结论。
- 截图数据有识别不确定性时，先列出识别结果请用户确认关键数字，再展开分析与建议。
- 标注所有数据（持仓、行情、财务、消息）的日期，避免用过期信息下结论。
- 单只股票的独立深度分析也可直接用 `stock-analysis` skill；组合场景下个股深挖内建在本 skill 流程中。
